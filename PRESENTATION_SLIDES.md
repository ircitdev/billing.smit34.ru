## Слайд 2 — Проблема

**Заголовок:** Устаревшие биллинговые системы тормозят бизнес

- Закрытый код, нет возможности доработки
- Зависимость от одного вендора и его ценовой политики
- Медленный интерфейс на устаревших технологиях (Firebird, Delphi)
- Отсутствие API для интеграций
- Сложное развёртывание и масштабирование

> Скриншот: Старый интерфейс Carbon Billing 4 (для контраста)

---

## Слайд 3 — Решение

**Заголовок:** СмИТ Биллинг 1.0 — новый подход

- Открытая архитектура на Python + Django + PostgreSQL
- Современный веб-интерфейс (AdminLTE 3, Bootstrap 5)
- REST API для любых интеграций
- Docker-контейнеризация: развёртывание за 15 минут
- RADIUS-авторизация (Internet, VoIP, IPTV)

> Скриншот: Главная страница биллинга — список абонентов с зелёными/красными статусами

## Слайд 11 — Docker-развёртывание

**Заголовок:** Запуск за 15 минут

```bash
git clone ...
cp .env.docker .env
docker-compose -f docker-compose.prod.yml up -d
```

- PostgreSQL, Redis, Web, Celery, FreeRADIUS — всё в контейнерах
- Готовые конфигурации для dev и prod
- Автоматические миграции при старте
- Nginx reverse proxy с раздачей статики
